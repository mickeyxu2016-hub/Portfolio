The Business Problem

Komatsu's mining equipment generates enormous volumes of operational data — but extracting insights from it required technical SQL expertise, creating a bottleneck between the data and the business users who needed it most.

The core challenge: How can we provide instant, reliable data insights through an AI-powered interface that any analyst or operations manager can use, without writing queries?

Data Overview

We worked with extensive industrial datasets generated by Komatsu mining machines across three categories:

Equipment Health
Error episodes, event codes, and shift-level damage and abuse metrics tracking machine condition over time.
Operations Productivity
Shift reports covering dig hours, idle hours, payload amounts, and 10-minute utilization averages.
Dig Cycle Performance
Detailed timing, payload measurements, and machine positional metrics captured during active operation.

Our Solution

The final deliverable is an intelligent chatbot that answers advanced data questions — such as "What is the average productivity across the 4800 lineup?" — with at least 80% accuracy. We used a Semantic Data Model powered by NLP to bridge the gap between plain English and raw SQL queries, eliminating the need for technical expertise at the point of query.

This design choice also prevents LLM hallucination — by anchoring responses to a defined semantic layer rather than open-ended generation, the answers remain factually grounded in the actual data warehouse.

Technologies Used

Snowflake Cortex
Central data warehouse hosting secure LLMs and defining the semantic models that ensure reliable, hallucination-free query generation.
Streamlit
Rapid front-end development of the AI assistant interface, with seamless Snowflake integration and a clean, dynamic UI for end users.

Project Overview

This project designed an end-to-end AWS cloud data architecture to analyze corporate bankruptcy risk using financial ratios. The pipeline handles data from its raw landing in S3, through ELT operations in Glue and Redshift, all the way to predictive modeling in SageMaker — reflecting a real-world analytics engineering workflow where clean, reliable data is the foundation of every good model.

The Data Pipeline Challenge

The project sourced data from four files containing financial stats, ratios, bankruptcy flags, and ten unlabeled company profiles. Mixed file formats (CSVs and pipe-delimited text) reflected the messy reality of production data engineering — nothing comes in clean.

  • Initial Architecture: Raw data was landed into an Input-data bucket on Amazon S3. AWS Glue was configured to perform inner joins across the ratio data files.
  • The Crawler Problem: The Glue crawler inferred inconsistent data types and encountered untrimmed join keys, causing nearly half of all rows to fail the join — a serious data integrity issue.
  • Pivoting to ELT in Redshift: I shifted to an ELT strategy, copying all source files directly into Redshift and using a custom SQL query (CREATE TABLE combined_data) to enforce schema strictness, trim join keys, and fully restore expected row counts.

Predictive Modeling with SageMaker

The cleaned master dataset was loaded from Redshift into SageMaker Canvas for model training. The 10 unlabeled company profiles were kept in a separate Predict-data S3 folder to maintain data governance and auditability throughout the process.

  • Model Training: Using an 80/20 train/test split, SageMaker Canvas produced a model with 92.522% accuracy on bankruptcy prediction — a robust result for financial risk classification.
  • Exploratory Insights: Correlation analysis showed that liquidity and profitability metrics are the strongest indicators of company health, while high variance in liability metrics powerfully predicts bankruptcy risk — findings directly useful for credit analysts.
  • Batch Inference: Predictions on the 10 target companies produced confidence scores between 85% and 97%, giving stakeholders reliable, high-confidence risk classifications they can act on.
